# Bebop CLI — Design Spec (2026-07-09)

> "Your kitchen, your ship, your cut." — Warm Cosmo-Noir, a deterministic guard OS.
> This doc is the *design authority* for the native Rust/WASM rewrite of the Bebop CLI.
> It replaces the hand-written TypeScript layer per the operator's standing directive:
> **"replace everything possible with kernel/rust/wasm — do not use TS at all."**

## 0. Why a native rewrite (not a TS wrapper)

The old CLI (`bebop.ts`, ~50 TS modules) is a Node/tsx process that loads a WASM
kernel for the math core. That left a large, un-verifiable-by-Rust surface (TS test
suites, runtime glue) and a node dependency at the core of a "sovereign" agent.

New architecture — **one binary, two faces**:

```
bebop (native rust binary, ratatui TUI)
 ├── lib core   = portable Rust logic (copilot, multipilot, vault, outfit,
 │               field, memory, governor, knowledge, dual-track)
 ├── wasm bindgen (optional feature) = same lib, compiled to bebop_core.wasm
 │               for the web/build pipeline
 └── TUI       = ratatui frontend (launch anim, frames, prompts)
```

- **No hand-written TS.** Every module that was `src/*.ts` becomes a Rust `mod`
  with `#[cfg(test)]`/`#[test]` coverage. The TS test suites are deleted.
- **Same Rust source compiles to native + wasm** (one implementation, two targets).
- Brand contract (`outfit`) becomes a Rust `const OUTFIT` — single source of truth,
  read by both the TUI and the README/wiki (which link to it, never re-declare).

## 1. Competitive teardown (what agents get right, what Bebop closes)

Researched live docs: Claude Code (code.claude.com/docs), OpenCode (opencode.ai),
Hermes Agent (hermes-agent.nousresearch.com/docs), Ratatui (ratatui.rs).

| Capability | Claude Code | OpenCode | Hermes | **Bebop (new)** |
|---|---|---|---|---|
| Project memory file | `CLAUDE.md` | `AGENTS.md` | `AGENTS.md`/`SOUL.md` | `AGENTS.md` + `OUTFIT` (brand contract) |
| Skills / procedural memory | skills | — | **self-creating skills + auto-improve** | skills (port) + self-maintain |
| Multi-agent | `claude agents` bg sessions | **multi-session** | delegate + profiles | **Multipilot** (N distinct pilots + synthesizer, field-gated) |
| Hooks / pre/post | hooks | — | cron + webhooks | hooks (port) |
| Share a session | — | **share-links** | gateway platforms | node + sync (P2P) |
| Privacy / offline | cloud | **privacy-first, no storage** | runs anywhere | **air-gapped core, no RNG/Date** |
| Customization | output styles | — | **skins/voice/personality/SOUL.md** | **OUTFIT: looks + narration + patrons** |
| Animation / identity | none | none | none | **red spaceship launch (ratatui)** |
| Guard / determinism | permission modes | — | approvals | **Guard OS: deny-on-red, Verified-by-Math** |
| Math proven core | — | — | — | **graph-PDE field + Kalman + GOAP (Rust)** |

**Gaps the others leave open that Bebop owns:**
1. **A face.** Claude/OpenCode/Hermes are faceless. Bebop has an *outfit* — a
   versioned identity (ship mark, palette, voice, sponsor line) that the CLI renders.
2. **A deterministic, math-proven arbiter.** Field-cost as a hard veto over PDDL
   plans. Nobody else ships a physics gate on the planner.
3. **Multipilot as default.** Not "spawn subagents" — a standing crew that argues and
   a synthesizer that decides, gated by the field. Other tools treat fan-out as opt-in.
4. **Offline-first sovereign node.** No RNG, no Date at runtime, encrypted-at-rest
   identity (vault). The agent can't lie about time or roll dice.
5. **Launch ritual.** A red spaceship animation on boot — a *moment*, not a spinner.

## 2. Brand contract (the single source of truth)

From `docs/design/dowiz-brand/BRAND-BIBLE.md` (canon) → encoded as Rust `OUTFIT`:

- **Warm Cosmo-Noir** — Cowboy Bebop × cosmo-gothic × Ukrainian irony.
- **"Hybrid is a feature, not a bug."** (the creed)
- **One meaningful color per view** (brand law): the launch uses **blood-red `#E0543E`**
  on the **void `#12100E`**; data/success/identity uses **teal `#46B0A4`**; bone
  text `#F2E9DB`; amber `#E8A544` for warnings.
- The ship mark `◈` (cold teal diamond, the machine's eye).

```rust
pub const OUTFIT: Outfit = Outfit {
    version: "1.0.0",
    era: "2026-07-09 — the new outfit",
    name: "Bebop",
    tagline: "your kitchen, your ship, your cut.",
    creed: "Hybrid is a feature, not a bug.",
    palette: Palette { teal: 0x46B0A4, bone: 0xF2E9DB, void: 0x12100E, amber: 0xE8A544, blood: 0xE0543E },
    sigil: '◈',
    narration: Narration::Bebop,
    home: "https://github.com/SyniakSviatoslav/bebop",
};
```

`bebop outfit` prints this contract; `bebop init --preset bebop` writes a profile that
reads from it.

## 3. The red spaceship launch animation (ratatui)

Design (TTY-gated, skippable via `NO_ANIM=1` or non-TTY):

- **Phase 0 — ignition (void):** screen fills with `#12100E`. A single amber spark
  blinks at center-bottom (the thruster).
- **Phase 1 — burn:** a column of `#E0543E` (blood-red) cells rises from the spark,
  with a slight per-frame jitter (deterministic LCG seeded by a *compile-time* constant —
  NOT `Date`/`rand` — to honor the air-gapped constraint). The trail leaves a fading
  afterglow (alpha ramp from blood→void).
- **Phase 2 — lift:** the ship glyph (an ASCII/Unicode chevron `ᐅ` or the `◈` sigil)
  ascends along the trail, nose-up, with two side fins drawn as `╱ ╱`.
- **Phase 3 — settle:** ship reaches mid-screen, the `OUTFIT` banner fades in in teal,
  the creed prints in bone. Animation ends; TUI frame takes over.

Determinism note: the only "randomness" is an LCG `x = x*1664525 + 1013904223` with a
const seed. No `std::rand`, no `chrono::now`. This keeps the launch provably
reproducible (a `#[test]` asserts the same seed → same frame buffer).

## 4. Customization axes (the "fascinating" surface)

Per Dowiz brand + Hermes `SOUL.md`/skins precedent, Bebop exposes three axes, all
stored in `~/.bebop/profile.toml` (native, no JS):

1. **looks** — palette override (must keep WCAG-safe pairings). Default = OUTFIT.
   `bebop init --looks blood` swaps the accent to `#E0543E` for the whole session.
2. **narration** — `bebop | plain | sarcastic | corporate-killer` (the dry
   Ukrainian-irony voice is `bebop`). Drives the copilot's one-liners.
3. **patrons** — sponsor line (`home:`), shown in the banner. Lets a fork re-skin
   the ship without forking the core.

`bebop init` interactively (or via `--json`) sets all three; `bebop outfit` shows the
resulting contract. This is the "make it yours" hook that Claude/OpenCode lack.

## 5. Command surface (native)

```
bebop                      # interactive TUI (launch anim → frame)
bebop boot                 # guard self-test (Verified-by-Math; refuse if gates can't go RED)
bebop init [--preset X|--json {...}|--looks Y|--force]
bebop outfit               # print the cosmo-noir identity contract
bebop status               # backend rotation + guard state
bebop dispatch "<task>"    # multipilot (DEFAULT) gated by field arbiter
bebop multipilot "<task>"  # explicit fan-out (--n N --no-field)
bebop route <class>       # token-router decision
bebop recall <q>          # living-knowledge §0·GP retriever
bebop node [--pass X]     # boot encrypted-at-rest node identity (vault)
bebop map [module]        # render real import graph (SVG)
bebop diagrams             # regenerate every visual
bebop mcp                 # MCP server over stdio
bebop help
```

Slash commands inside the TUI mirror the old `/help /status /model /clear /compact
/resume /review /skills` set (Claude Code parity) + `/outfit /dispatch /multipilot`.

## 6. Migration map (TS → Rust, with test porting)

| Old (TS) | New (Rust mod) | Test port |
|---|---|---|
| `src/vault.ts` (XChaCha20-Poly1305 + scrypt) | `core::vault` (`src/vault.rs`: ML-KEM-768 ⊕ X25519 + ML-DSA-65 ⊕ Ed25519, argon2id + chacha20poly1305) | `#[test]` wrong-pass throws, tamper rejected, hybrid sig required |
| `src/outfit.ts` | `core::outfit` (`const OUTFIT`) | `#[test]` palette WCAG + version semver |
| `src/copilot.ts` / `multipilot.ts` | `core::copilot` / `core::multipilot` | distinct-pilot invariant, field override blocks |
| `src/loop.ts` | `core::loop` | react trace visible, denial in trace |
| `src/knowledge.ts` / `memory.ts` | `core::knowledge` / `core::memory` | recall returns real payload, noise floor |
| `src/governor.ts` | `core::governor` (PID) | authority up on approve, down on reject |
| `src/integration/analytics/*` | `core::analytics::*` | N1–N8 + field planner RED+GREEN |
| `rust-core/src/lib.rs` | `core::field` (wasm + native) | 16→ keep + add native |
| `src/theme.ts` / `voice.ts` | `core::theme` / `core::voice` | palette + narration axes |

TS test files (`*.test.ts`) are **deleted**; their assertions are re-expressed as
`#[test]` in the corresponding Rust mod. The `verify-doc-claims` and
`guardrail-falsifiable-proof` scripts are ported to a Rust `#[test]` harness
(`core::doc_claims`) so the "every test can go RED" invariant survives the TS purge.

## 7. Verification (Verified-by-Math, survives the rewrite)

- `cargo test` → all Rust mod tests green (the 433 TS assertions now live here).
- `cargo build --release --target wasm32-unknown-unknown` → `bebop_core.wasm` clean.
- `bebop boot` → guard self-test prints `certified`, exits 1 on any red-line failure.
- `bebop outfit` → prints `OUTFIT` (teal), asserts same bytes as the const.
- Launch determinism: `#[test]` seeds LCG → identical frame buffer (reproducible).
- Doc-gate: `cargo test doc_claims` asserts README/wiki claims match `OUTFIT` + counts.
