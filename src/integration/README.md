# bebop integration layer — bleeding-edge tools reverse-engineered

All modules are **deterministic** (Float64 / integer math, no RNG, no `Date`, no network at
unit-test time), **falsifiable RED+GREEN tested**, and **typecheck-clean** under the repo's
own `tsconfig`. Verified: `node --test --import tsx 'src/**/*.test.ts'` → 265 pass / 0 fail;
`npx tsc --noEmit` → exit 0.

## How the integration is wired
- Each module is **feature-flag-free and pure**: import + call. No global side effects.
- The Sovereign Node can compose them: Zenoh = transport, TigerBeetle = operational ledger,
  zkVM `decide` = authenticated money boundary, Active Inference = policy selection for the
  agent loop, optical = optional co-processor kernel, FinalSpark = out-of-band wetware signal.

## Verification per module (RED+GREEN)
See each `*.test.ts`. Every property pairs a GREEN (passes on valid input) with a RED
(must FAIL on invalid input: gain mask, NaN preference, non-conserved ledger, tampered journal).

## Honest gaps (no false-green)
- `zenoh`: real client is optional (native binary); the module is a dependency-free semantic
  twin so CI stays hermetic. Flip `USE_REAL_ZENOH` when a Zenoh router is present.
- `zkvm`: the Rust guest compiles + its `cargo test` determinism passes, but **STARK proving**
  needs the `risc0` toolchain / prover; verification is exercised via the TS verifier on the
  same digest. Proving is best-effort and documented as such.
- `finalspark`: the deterministic LIF stub is the test surface; the live organoid adapter is a
  documented remote call (non-deterministic bio substrate — kept out of the deterministic core).
